# Contributing to Localman

First off, thank you for considering contributing to Localman! It's people like you that make Localman such a great tool.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issue Tracker.
- Check if the issue has already been reported.
- If not, create a new issue. Please include:
    - A clear and descriptive title.
    - Steps to reproduce the bug.
    - Your OS and version of Localman.
    - Any relevant logs or screenshots.

### Suggesting Enhancements
- Open a GitHub Issue with the label `enhancement`.
- Describe the feature you'd like to see and why it would be useful.

### Pull Requests
1. **Fork the repo** and create your branch from `main`.
2. **If you've added code** that should be tested, add tests.
3. **If you've changed APIs**, update the documentation.
4. **Ensure the test suite passes**.
5. **Make sure your code lints**.
6. **Submit the PR**!

## Development Setup

See the [README.md](README.md) for detailed instructions on setting up your local environment.

## Style Guidelines

### Git Commit Messages
- Use the present tense ("Add feature" not "Added feature").
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...").
- Limit the first line to 72 characters or less.

### TypeScript / React
- Follow the existing project structure in `src/`.
- Use functional components and hooks.
- Ensure all new components are documented.

### Rust
- Use `cargo fmt` before submitting.
- Follow standard Rust naming conventions.

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.

## Questions?
Feel free to open an issue or reach out to the maintainers!
